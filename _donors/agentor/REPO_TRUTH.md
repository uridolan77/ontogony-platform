# Repository truth — current behavior

This document states **what the code and HTTP surface actually do today**, so operators and integrators do not mistake roadmap or internal components for default production behavior.

## Public agent runs

- **`POST /api/v1/agent-runs`** flows through **`IAgentRunOrchestrator`** (`AgentRunOrchestrator`). Clients select execution with **`planId`**, **`recipeId`**, **`toolKey`**, **`skillKey`**, and/or **`mode`** (see `StartAgentRunRequestDto`). **`RunExecutionMode.LegacyFakeTool`** (or **`Agentor:PublicRuns:TreatMissingExecutionSelectorAsLegacyFakeTool=true`**, the default) keeps the historical PR1 fake-tool path without specifying selectors.
- **Idempotency fingerprint (PR137.5)**: **`StartAgentRunFingerprint.Compute`** includes a canonical JSON segment for **`ToolInputPayload`** via **`JsonFingerprintCanonicalizer`** (sorted object keys, compact UTF-8) so equivalent structured payloads produce the same fingerprint regardless of JSON member order or insignificant whitespace.
- **Plan / recipe / skill** starts materialize a fresh `AgentPlan` and execute via **`SequentialAgentPlanExecutor`**. **Single-tool** paths (including Conexus, MCP, external-agent tool keys) use the shared governed single-tool driver (policy + pipeline) without embedding tool logic in `StartAgentRunHandler`.

## Real execution kernel

- **`SequentialAgentPlanExecutor`** is the coordinator for **plan-mode** public starts (and skill-wrapped ephemeral recipes). **Single-tool** public starts still use the same policy and **`IToolExecutionPipeline`** stack as plans, but do not route through the plan executor’s step loop.
- **Structured tool I/O (Phase 30 / PR121)**: **`ToolPayload`** (**JSON `body`**, optional **`schemaId`/`contentType`**, flat **`summary`**) is the execution envelope for **`ToolExecutionRequest`**/**`ToolCall`** persistence; **`ToolPayload.FromLegacyDictionary`**/**`ToLegacySummary`**/**`ToPolicyEvaluationDictionary`** bridge PR1-style flat maps. Integration DTOs (**`ModelCallRequestDto`/`ModelCallResultDto`**, **`ExternalAgentInvocationRequestDto`**, MCP HTTP invoke) serialize **`ToolPayload`** as JSON; audit export emits **`input`/`output`** objects with **`body`** + **`summary`** (redaction applies to nested JSON).
- **Production integration smoke (Phase 35 / PR138–PR142)**: **`Agentor:IntegrationSmoke`** (**`IntegrationSmokeOptions`**, per-family **`SmokeMode`**, **`SmokeTarget`** names for CLI filtering) drives the **`Agentor.IntegrationSmoke`** host; **`IntegrationSmokeConfigurationMerger`** overlays **`Agentor:Integrations:*:*:Mode`** for that process. **`IntegrationSmokeRunner`** exercises Athanor read paths and optional candidate submit (**`AllowAthanorWriteSmoke`**), Conexus **`CompleteAsync`** with declared budget + telemetry fields, MCP list/discover/invoke, and external-agent discover/invoke. **`IntegrationSmokeReportWriter`** emits **`integration-smoke-report.json`** / **`.md`** with **`IntegrationFailureRedaction`** on error text. Operator entry: **`scripts/run-integration-smoke.ps1`**; documentation **`docs/operator/integration-smoke.md`**.

## Evaluation science (Phase 32)

- **`EvaluationDatasetRegistry`** loads dataset JSON (validated against **`EvaluationFixtureRegistry`** entries), rejects duplicate ids / unknown fixtures, and selects cases by **`EvaluationCaseTag`** (**smoke**, **regression**, **review**, **external-agent**, **policy**, **queue**). **`EvaluationMetricSnapshot`** / **`EvaluationBaseline`** / **`EvaluationDeltaCalculator`** capture comparative metrics with stable JSON; **`EvaluationAggregateReportGenerator`** produces cross-run aggregates; **`EvaluationThresholdEvaluator`** evaluates **`evaluation-thresholds.json`** with stable reason codes (**`EVAL_THRESHOLD_*`**). **`EvaluationReportGenerator`** emits **`evaluation-report.md`**, **`evaluation-report.json`**, and **`evaluation-summary.csv`** (optional aggregate + threshold sections). **`PerformanceReportGenerator`** emits **`performance-report.md`**, **`performance-report.json`**, and **`performance-summary.csv`** alongside evaluation artifacts. CI publishes **`artifacts/evaluation`** via **`scripts/generate-evaluation-ci-artifacts.ps1`**.

## Policy scopes

- **`PolicyRule`** rules carry **`Scope`** (`Global`, `Tenant`, `Workspace`, `Project`, **`KnowledgeScope`**) plus optional **scope identifier** fields (`ScopeTenantId`, `ScopeWorkspaceId`, `ScopeProjectId`, `ScopeKnowledgeScopeId`) validated at construction.
- **`PolicyBundleRulesAdapter.ToProfileRules(bundle, AgentRunScope)`** includes only rules whose scope matches the run’s **`TenantId` / `WorkspaceId` / `ProjectId` / `KnowledgeScopeId`**. Conflicts merge by **specificity** (KnowledgeScope → Project → Workspace → Tenant → Global); at equal specificity, **Deny > RequiresReview > Allow** for tool-access rules.
- **`PolicyEvaluationRequest`** includes **`AgentRunScope`** so **`RuntimePolicyEvaluator`** builds the effective profile per tool call from the active bundle and the run’s identity.

## Persistence

- **`EfCoreAgentRunRepository.SaveAsync`** merges into the existing **`agent_runs`** row: upserts root scalars (including **`session_memory_json`**, **`human_review_decisions_json`**, and **`resume_cursor_json`** for **`PlanResumeCursor`** — including optional **`SkillContinuation`** for mid–skill-procedure human review), upserts **`agent_steps`** and nested **`tool_calls`** / **`policy_decisions`** by id, and **appends** new **`trace_events`** by id. Existing trace rows are **immutable**; a save that would rewrite payload raises **`AgentRunTraceImmutabilityException`**.
- **`agent_runs.aggregate_version`** is an optimistic-concurrency token (incremented on each successful save). **`AgentRun.PersistenceConcurrencyVersion`** is populated on **`GetAsync`** and refreshed after **`SaveAsync`**; a stale version yields **`AgentRunPersistenceConcurrencyException`** (mapped to **409 Conflict** on the HTTP surface). **`POST /agent-runs`** and other handlers that only create runs do not require a prior load.
- **Human review workflow (Phase 28; PR121.5 invariant hardening; PR122/122.5 application split)**: **`completed_at`** is **successful completion only** ( **`terminal_at` is null** on success). Review suspension uses **`review_requested_at`** / **`paused_at`**; failures and human rejection set **`terminal_at`** ( **`completed_at` stays null**). Loads normalize stale **`terminal_at`** on **`Completed`** rows. **`review_workflow_status`** persists **`HumanReviewWorkflowStatus`**. **`ApplyHumanReviewDecision`** APIs accept optional **`relatedPriorActorId`** for escalation chains. Post–PR122, resume/plan/trace coordination lives under **`src/Agentor.Application/HumanReview/`** (`HumanReviewDecisionApplicator`, `ReviewedToolContinuationService`, `PlanResumeOrchestrator`, `ReviewPolicyReevaluationService`, `ReviewTraceWriter`); escalated approve without governance approver maps to **403** `GovernanceApproverRequired` at HTTP boundaries. **Phase 34**: mid–skill-procedure **`RequiresReview`** persists **`PlanResumeCursor.SkillContinuation`**; post-approval execution continues inner tools under normal policy ( **`ResumeAfterApprovedHumanReview`** applies only to the approved inner call), then clears **`SkillContinuation`** before tail plan steps resume under the same orchestrator as multi-step tool plans.

## Authentication (Jwt / ASP.NET)

- **`UseAuthentication` / `UseAuthorization`** are enabled. **`/api/v1/*`** requires an authenticated principal (**`Agentor.Authenticated`** policy). **`GET /health`** stays anonymous.
- **Fake** mode registers **`Agentor.Fake`** authentication (stable dev principal). **Header** mode registers **`Agentor.Header`** (GUID in `X-Agentor-Actor-Id` by default) and issues **`HumanOperator`** only — **`HumanGovernanceApprover`** requires a JWT (or future header-role support). **Jwt** mode registers **`AddJwtBearer`** when **`Agentor:Auth:JwtAuthority`** is set; otherwise **`Agentor:Auth:JwtAcceptUnvalidatedBearerTokens=true`** enables **`Agentor.JwtUnvalidated`** (parses bearer JWTs **without** signature validation). That mode is **blocked outside Development/Test** unless **`Agentor:Auth:JwtAllowUnvalidatedTokensOutsideDevelopment=true`**.
- **OpenAPI**: **`GET /openapi/v1.json`** is mapped in **Development** and **Test**/**Testing** hosts by default; in **Production** it is **off** unless **`Agentor:OpenApi:Enabled=true`** (registered services always include OpenAPI generation; only route mapping is gated after configuration merge).
- **`ICurrentActorAccessor`** still maps **`HttpContext.User`** in Jwt mode (claims → `ActorRole`). See **`docs/security/auth-boundary.md`** and **`docs/security/AUTHORIZATION_MATRIX.md`**.

## Durable run queue worker

- **`RunQueueHostedService`** is registered as a **hosted singleton** but resolves **`IDurableRunQueue`**, **`IRunExecutionLeaseStore`**, and **`IAgentRunOrchestrator`** inside a **per-drain `IServiceScopeFactory` async scope**, so EF-backed scoped implementations share one **`DbContext`** for claim → execute → mark complete/failed. Drained items execute through **`StartAgentRunRouting`** + **`IAgentRunOrchestrator`** (not a separate fake-centered path). **`AddAgentorInfrastructure`** binds **`Agentor:PublicRuns`** so worker routing defaults match API configuration.
- **EF `run_queue_items`** rows persist **governance scope** (`tenant_id`, `workspace_id`, `project_id`, `knowledge_scope_id`) and **public orchestration selectors** (`execution_mode`, `recipe_id`, `plan_id`, `tool_key`, `skill_key`, **`tool_input_json`** for legacy string-keyed inputs, and **`tool_payload_json`** for structured **`ToolPayload`** v2) so queued work replays the same routing as inline **`POST /api/v1/agent-runs`** starts after dequeue (PR117.5 + Phase 33 **`tool_payload_json`**).
