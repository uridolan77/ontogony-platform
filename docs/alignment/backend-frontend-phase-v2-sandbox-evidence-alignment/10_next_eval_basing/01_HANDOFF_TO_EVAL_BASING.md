# Handoff to Eval-Basing

This package intentionally stops before eval-basing.

## What eval-basing should consume from this bridge

- audit bundle evidence shape
- event timeline rendering
- side-effect ledger rows
- capability metadata
- OpenAPI provenance
- compatibility reports
- frontend evidence panels

## Eval-basing first moves

### EVAL-001 — Ontogony neutral evaluation contracts
Add mechanical evaluation records, no product semantics.

### EVAL-002 — Allagma eval run metadata
Add evaluation profile, baseline run ID, topology metadata, quality signals.

### EVAL-003 — Kanon topology/policy authorization
Allow/deny topology modes and validation modes.

### EVAL-004 — Conexus route decision evidence
Capture model route decision IDs, fallback chain, capability profile.

### EVAL-005 — First baseline comparison suite
Compare single workflow baseline against any more complex topology.
