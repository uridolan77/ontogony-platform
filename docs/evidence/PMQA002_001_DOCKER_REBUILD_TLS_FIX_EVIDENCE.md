## 1) Reproduction

Reproduced with:

```powershell
cd C:\dev\ontogony-platform
.\docker\local-working-system\scripts\start-local-working-system.ps1 -Build
```

Observed in Docker build restore stages (`allagma-api`, `kanon-api`, `conexus-api`):

- `error NU1301: Unable to load the service index for source https://api.nuget.org/v3/index.json`
- `The remote certificate is invalid because of errors in the certificate chain: PartialChain`

## 2) Root cause isolation

Inside `mcr.microsoft.com/dotnet/sdk:9.0-bookworm-slim`, direct TLS probe to NuGet failed:

- `curl: (60) SSL certificate problem: unable to get local issuer certificate`

`openssl s_client` showed `api.nuget.org` was being re-signed locally by:

- `CN=Avast Web/Mail Shield Root, O=Avast Web/Mail Shield, OU=generated by Avast Antivirus for SSL/TLS scanning`

Conclusion:

- Not a NuGet source misconfiguration.
- Not an upstream .NET SDK image defect by itself.
- This machine uses local TLS interception; Windows trusts that root CA, Linux Docker build containers do not by default.

Classification: **local machine trust-store / TLS interception CA trust gap in Docker build context**.

## 3) Narrow safe fix implemented

No TLS verification disablement and no certificate material committed.

### Changes

- `allagma-dotnet/Dockerfile`
- `kanon-dotnet/Dockerfile`
- `conexus-dotnet/Dockerfile`
  - Added optional build arg `EXTRA_CA_CERT_BASE64`.
  - If provided, cert is installed into `/usr/local/share/ca-certificates/extra-root-ca.crt` and `update-ca-certificates` is run.

- `ontogony-platform/docker/local-working-system/docker-compose.yml`
  - Passed `EXTRA_CA_CERT_BASE64: ${DOCKER_EXTRA_CA_CERT_BASE64:-}` to all three .NET build services.

- `ontogony-platform/docker/local-working-system/scripts/start-local-working-system.ps1`
  - Added automatic NuGet TLS probe from SDK container during `-Build`.
  - If probe fails, script detects NuGet issuer, finds matching trusted root in Windows cert store, exports it in-memory as PEM, and sets process env `DOCKER_EXTRA_CA_CERT_BASE64` for compose build.
  - Added `-SkipFrontend` mode for backend-only deterministic rebuild/verification.
  - Added `-DisableAutoCaInjection` switch for explicit opt-out.

- `ontogony-platform/docker/local-working-system/.env.example`
  - Added documented optional `DOCKER_EXTRA_CA_CERT_BASE64` placeholder.

- `ontogony-platform/docker/local-working-system/README.md`
  - Documented automatic CA injection behavior and backend-only startup command.
