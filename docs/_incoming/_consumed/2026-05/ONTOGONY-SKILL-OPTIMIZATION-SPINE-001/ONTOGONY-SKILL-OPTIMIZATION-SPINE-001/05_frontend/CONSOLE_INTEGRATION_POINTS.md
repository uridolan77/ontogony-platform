# Console Integration Points

## Existing pages likely to link Skill Lab

Add links, not clutter, from:

```text
Home/System overview
Allagma run detail
Conexus model call detail
Kanon decision detail
Evidence Spine graph/detail
Agent interaction workbench
```

## Link labels

Use concise labels:

```text
Skill version
Skill gate
Skill edits
Deployment binding
Optimization run
```

## Evidence Spine integration

Where a SkillVersion appears in Evidence Spine, show a compact node:

```text
SkillVersion
v0.1.1 · accepted · deployed local
Gate +0.17
```

Click opens Skill Detail, not a raw JSON dump.

## Operator dashboard metric

Optional first-slice metric:

```text
Active skill deployments: N
Rejected candidate edits: N
Last accepted skill gate: +delta
```

## Avoid console overload

The Skill Lab should reduce existing console clutter. Do not add long explanatory blocks to every page. Centralize explanation in a concise help drawer or docs link.
