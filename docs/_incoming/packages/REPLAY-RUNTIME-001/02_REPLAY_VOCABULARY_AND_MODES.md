# 02 — Replay vocabulary and modes

## Core rule

Every replay path must be honest about what it is doing. The mode is not cosmetic. It is part of the truth contract.

## Replay modes

### `exact_replay`

Re-executes the original computation with the same inputs, same deterministic dependencies, same versioned code/config/data snapshot, and no nondeterministic external dependency.

Allowed only when all of the following are true:

- input payload is preserved or reconstructable losslessly;
- code/config/ontology/model alias/provider behavior snapshot is available;
- no real external provider/tool call is required, or the original output was captured and replayed without re-execution;
- random/time sources are fixed or irrelevant;
- resulting output can be meaningfully compared byte-for-byte or semantically with defined tolerances.

Default assumption: unavailable unless proven.

### `deterministic_simulation`

Runs a deterministic simulation using preserved evidence and controlled fake/local dependencies. It may execute code paths, but not real providers or real external tools.

Typical examples:

- governed fake Allagma run replay;
- Kanon semantic decision replay against pinned ontology/source snapshots;
- Conexus route decision replay against pinned routing config;
- fake-provider model-call simulation.

### `dry_run`

Executes planning/eligibility/routing logic without committing side effects and without calling real external providers/tools.

Dry-run can answer:

- What would Allagma attempt?
- Which Kanon policy/gate would apply?
- Which Conexus route would be chosen?
- Which provider would be selected?
- Which operation would be blocked?

Dry-run must not claim to reproduce the original output.

### `reconstructed`

Builds a timeline or result view from recorded evidence without re-executing the original operation.

Typical examples:

- Allagma run event stream replay as a timeline;
- Agent Interaction replay from stored events;
- Trace/correlation evidence graph reconstruction.

### `evidence_only`

Returns the original captured evidence, replay manifest, source attempts, hashes, links, and unavailable reasons. No re-execution and no simulation.

This is the current Allagma replay manifest shape.

### `unavailable`

Replay cannot be performed in any meaningful mode for the requested target. The response must explain why.

### `unsafe`

Replay could technically be attempted but is blocked by safety policy because it would require real external provider/tool execution, unsafe side effects, missing consent, missing confirmation, or missing allowlist.

In the API, `unsafe` is represented as `mode = unavailable` with `unavailableReason.category = safety_blocked` unless a separate `safetyStatus` field is present.

## Evidence terms

### Original evidence

The evidence captured during the original run/decision/model call/provider attempt/trace.

### Replay evidence

Evidence generated by the replay workflow, including replay request, eligibility, service attempts, result bundle, delta, safety posture, and export metadata.

### Replay delta

A structured comparison between original evidence and replay evidence. Deltas can compare status, route decision, semantic decision, model output fingerprint, provider selected, token usage, cost estimate, human gate result, timeline order, or unavailable/skipped reasons.

### Replay verdict

The final high-level result:

- `matched`
- `matched_with_expected_drift`
- `diverged`
- `blocked`
- `inconclusive`
- `failed`

### Replay root identifier

The identifier the operator begins with, such as:

- `allagma.run:{runId}`
- `allagma.audit_bundle:{runId}`
- `allagma.interaction_stream:{runId}`
- `kanon.decision:{decisionId}`
- `kanon.semantic_plan:{planId}`
- `kanon.provenance:{provenanceId}`
- `conexus.model_call:{modelCallId}`
- `conexus.route_decision:{routeDecisionId}`
- `conexus.provider_attempt:{providerAttemptId}`
- `platform.trace:{traceId}`
- `platform.correlation:{correlationId}`
- `evidence_spine_bundle:{bundleId}`

## Required labels in UI

Every replay result panel must show:

- Replay mode.
- Safety posture.
- Whether real providers/tools were blocked or disabled.
- Source evidence coverage.
- Unavailable/skipped reasons.
- Evidence Spine links.
- Replay verdict.
