# EVAL-EVIDENCE-QUALITY-001

Cursor implementation package for tightening the Ontogony console evaluation/evidence-quality experience.

This package is intentionally scoped to the operator console and frontend evidence presentation first. Backend/API changes are allowed only as documented follow-ups unless the current repo already exposes the required fields/routes.

Primary area:
- Allagma → Evaluations dashboard
- Baseline comparison entry points
- Evaluation row evidence links
- Evidence Spine linkage from evaluations/runs
- Page-level quality summaries and trend honesty

Core thesis:

> The evaluation dashboard must stop looking like broad quality proof when it is only a current-page list of sparse live evaluation rows. Every quality claim must expose its evidence basis, metadata completeness, and drilldown path.
