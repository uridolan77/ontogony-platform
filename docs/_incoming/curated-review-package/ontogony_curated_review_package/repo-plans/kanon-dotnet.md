# kanon-dotnet plan

## Priority

Kanon should prove durable local semantic authority and produce evidence packs for its already-substantial workflow surface.

## Keep

- Ontology lifecycle.
- Source bindings.
- Semantic plans.
- Canonical fact resolution.
- Action policy and human gates.
- Decision provenance/replay.
- Domain-pack management.
- Conexus assistance as advisory and disabled/mock/local by default.

## Implement next

1. `KANON-POSTGRES-LOCAL-001`
2. `KANON-EVIDENCE-001`
3. `KANON-AUTH-LOCAL-001`
4. `KANON-API-MODULAR-001`
5. `KANON-CONEXUS-ASSIST-001`
6. `KANON-DOMAINPACK-GOV-001`

## Notes

The priority is not to add real provider assistance. The priority is deterministic local proof, service-token mode, better evidence, and clearer endpoint organization.
