# SYSTEM-TRUTH-001 Cursor Package

Purpose: make the Ontogony operator console truthful, consistent, and evidence-backed for the local alpha stack.

This package focuses only on the **System Truth** workstream:

- normalized `/health` contracts
- normalized `/ready` contracts
- service version/build metadata
- Conexus readiness explanation
- compatibility summary artifact
- separation of liveness/readiness/contract/compatibility/release-readiness
- fixture/demo/generated data-source semantics
- operator Home / Settings / Topology truth-hardening
- route/readiness status vocabulary cleanup
- a smoke harness that proves the console is reading live facts, not optimistic copy

It intentionally does **not** implement deeper Evidence Spine graph completion, Kanon source-binding polish, Agent Interaction live timeline, or Evaluation metadata. Those belong to later workstreams.
